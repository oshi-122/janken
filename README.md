# janken

このリポジトリは現在、初期化直後の最小構成です。

## 現在の構成

- `.git/`: Git の履歴や設定（通常は直接編集しない）
- `.gitkeep`: 空ディレクトリ/最小状態を維持するためのプレースホルダ

現時点では、アプリケーションコード・設定ファイル・テストコードはまだ追加されていません。

## 新規参加者向けの重要事項

1. **まず技術スタックを決める**
   - Web アプリなのか CLI ツールなのか
   - 言語（TypeScript/Python/Go など）
   - パッケージマネージャ（npm/pnpm, pip/uv など）

2. **初期ディレクトリ設計を合意する**
   - 例: `src/`, `tests/`, `docs/`, `scripts/`
   - 例: `README.md`, `CONTRIBUTING.md`, `.editorconfig`, `.gitignore`

3. **開発ルールを先に決める**
   - ブランチ戦略（main + feature ブランチなど）
   - コミット規約（Conventional Commits など）
   - CI（lint/test を PR 時に自動実行）

## 次に学習・整備すると良い順序

1. **README の拡充**
   - セットアップ方法、起動方法、テスト実行方法
2. **最小アプリの雛形作成**
   - `src/` とエントリーポイントを追加
3. **テスト基盤の導入**
   - 単体テスト 1 本を先に追加
4. **Lint/Formatter の導入**
   - チーム全員のコードスタイルを統一
5. **CI の導入**
   - push/PR ごとに lint + test を回す

## 期待される次のマイルストーン

- M1: 技術選定とディレクトリ構成の合意
- M2: Hello World レベルの実行可能コード
- M3: テストと CI が通る最小開発サイクルの完成
